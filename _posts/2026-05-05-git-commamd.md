Git는 프로젝트의 변경 이력을 저장하고 관리하는 버전관리 시스템이다.<br>
그러므로 프로젝트를 진행하다 보면 파일을 여러번 수정하거나 실수로 잘못 수정되는 상황이 발생할 경우 변경 이력 관리와 복구할 수 있고, 협업을 진행할 때, 변경 내용을 분리하여 합칠수 있는 브랜치와 병합 개념으로 인하여 많이 사용한다.
<br>
###Git 기본 명령어 및 로컬 작업
`git commit`: 프로젝트의 특정 시점을 저장하는 기록으로 현재 변경 사항을 저장소에 기록한다.<br>
`git commit —amend`: 가장 최근에 저장한 것을 수정한다.<br>
`git branch`: 작업 흐름을 나누는 가지로브랜치를 생성하거나 확인한다.<br>
