Git는 프로젝트의 변경 이력을 저장하고 관리하는 버전관리 시스템이다.<br>
그러므로 프로젝트를 진행하다 보면 파일을 여러 번 수정하거나 실수로 잘못 수정되는 상황이 발생할  변경 이력 관리와 복구할 수 있고, 협업을 진행할 때, 변경 내용을 분리하여 합칠 수 있는 브랜치와 병합 개념으로 인하여 많이 사용한다.
<br>
### Git 기본 명령어 및 로컬 작업
`git commit`: 프로젝트의 특정 시점을 저장하는 기록으로 현재 변경 사항을 저장소에 기록한다.<br>
`git commit —amend`: 가장 최근에 저장한 것을 수정한다.<br>
<br>
`git branch [브랜치명]`: 작업 흐름을 나누는 가지로브랜치를 생성하거나 확인한다.<br>
`git branch -f [브랜치명] [위치]`: 브랜치를 특정 커밋으로 강제로 이동한다.<br>
<br>
`git merge [브랜치]`: 현재 브랜치에 지정한 브랜치를 병합한다.<br>
<br>
`git rebase [브랜치명]`: 현재 브랜치를 다른 브랜치 위로 옮긴다. <br>
`git checkout -i [위치]`: 커밋들을 직접 수정하거나 합치거나 삭제할 수 있는 편집 모드이다.<br>
<br>
### HEAD와 상대 참조
HEAD는 현재 작업 중인 위치를 가리키는 특수한 포인터이다. 특정 커밋을 직접 가리키는 경우에는 분리된 HEAD 상태가 된다.
`git checkout [브랜치명]`/`git switch [브랜치명]`: 현재 작업 중인 HEAD를 해당 브랜치로 이동시킨다.<br>
`git checkout -b [브랜치명]`: 새로운 브랜치를 생성하고 동시에 그 브랜치로 이동한다.<br>
<br>
`^`(캐럿): 한 단계 위 커밋으로 이동한다. <br>
`~[숫자]`(틸데): 지정한 숫자만큼 위 커밋으로 이동한다.<br>
<br>
### 작업 옮기기
`git cherry-pick <commit1> <commit2>`: 지정한 커밋들을 현재 브랜치에 그대로 가져와 적용한다.<br>
`git rebase -i[위치]`: 커밋의 순서를 바꾸거나, 합치거나, 삭제할 수 있는 편집 모드를 연다.<br>
<br>
### Push &Pull(원격 저장소)
`git clone [URL]`: 원격 저장소의 전체 내용을 내 로컬 컴퓨터로 복제해 온다.<br>
`git getch`: 원격 저장소의 최신 변경 사항을 확인하고 로컬에 다운로드 한다. 단, 재 작업과 병합되지 않는다.<br>
`git pull`: 원격 저장소의 데이터를 가져와서 내 로컬 브랜치에 즉시 병합한다.<br>
`git push`: 내 로컬에서 작업하고 커밋한 내용을 원격 저장소에 업로드한다.<br>
### 고급 원격 저장소
`git fetch origin [소스]:[목적지]:`: 원격의 특정 브랜치만 가져와서 로컬의 특정 브랜치에 반영한다.<br>
`git push origin [소스]:[목적지]:`: 내 로컬의 특정 작업을 원격 저장소의 다른 브랜치 이름으로 밀어 넣는다.<br>

### 느낀 점
